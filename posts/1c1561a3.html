1. 认识 Vuex

Vuex 是一个专为 Vue.js 应用程序开发的状态管理模式。它采用集中式存储管理应用的所有组件的状态，并以相应的规则保证状态以一种可预测的方式发生变化。

看得懂吗，看不懂就对了。

所谓的状态管理，就是把需要多个组件共享的变量全部存储在一个对象里面，将这个对象放在顶层的 Vue 实例中，让其他组件可以使用。常见的状态例如用户的登录状态、用户名称、头像、地理位置信息等等。

在之前的学习中，可以体会到 Vue 帮助我们很好地管理了单界面的状态。以一个计数器的案例来举例，其中三个重要的组成部分分别为：

  • State：就是状态。（姑且可以当做就是 data 中的属性）
  • View：视图层，可以针对 State 的变化，显示不同的信息。
  • Actions：主要是用户的各种操作比如点击、输入等等，会导致状态的改变。
  • 在这个案例中，需要管理的状态就是 counter，对应了 State。
  • counter 当前的值需要显示在界面中，对应了 View。
  • 当用户点击按钮后，状态需要更新，对应了 Actions。

如果多个视图都依赖同一个状态，或者不同界面的 Actions 都想修改同一个状态，Vuex 就可以作为一个大管家，统一管理这些需要共同维护的状态。

下面会把这个案例按照多界面状态管理来改写。

3.2.4 响应规则

Vuex 中的 store 中的 state 是响应式的，当 state 中的数据发生改变时，Vue 组件会自动更新。这就要求我们必须遵守一些 Vuex 对应的规则：

  • 提前在 store 中初始化好所需的属性
  • 当给 state 中的对象添加新属性时，使用以下两种方式：
    • 使用Vue.set(obj,'newProp',value)
    • 使用新对象给旧对象重新赋值
  • 同样删除属性也有对应的方法Vue.delete(obj,'prop')
3.2.5 常量类型

这个在开发中用的非常多。简单来讲就是把「魔法字符串」定义成一个常量，这样可以保证在不同文件中使用同一个字符串的时候，不会因为敲错字符串导致程序出现问题。

有一个比较有意思的事情是，在学到这里的时候，看到很多弹幕对这种行为表示无法理解：

  • 「脱裤子放屁，多此一举」
  • 「复制粘贴不香吗？」
  • 「太麻烦了，我还是直接写字符串吧」

在以前的开发中，我就接触过这一思想，当时是新建一个常量类 Const，将所有的字符串都定义在其中，想要用的时候直接用类名去调用。甚至在安卓开发中，界面中所有的文字、颜色代码、字体大小等「资源相关」的东西都是在一个对应的文件中统一管理的。如果不这样做，编译器就会提示你「这里有魔法值」。

这样做的好处有很多，对于文字，可以更方便地做国际化；对于颜色代码，可以统一更改而不必在每一个用到这个颜色的界面都改；对于字体大小同理。所以这可不是「脱裤子放屁」的一件事。

对于这些弹幕，我真的觉得很可笑，这些人可能没有什么开发经验，没有写过几行代码，更没有考虑过作为一个程序员如何应对频繁变化的需求吧。

3.5 module

当应用变得复杂的时候，state 越来越多，不同模块的 state 都在一个 store 中，这样十分臃肿。为了解决这个问题，Vuex 允许定义 module，每个 mudule 拥有自己的s tate、mutations、actions、getters 等。
