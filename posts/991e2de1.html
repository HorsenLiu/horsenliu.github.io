0. 题外话

关于 JavaScript、TypeScript 或者 CSS 代码在缩进的时候到底是 4 个空格还是 2 个空格，查了很多文章和讨论，发现在前端开发中使用 2 个空格的人较多，并且很多开源项目和大型项目也是 2 个空格。

我个人的理解是，无论是几个空格，一个缩进就是一个 Tab，至于一个 Tab 到底是多少空格，前端开发习惯 2 个，后端开发习惯 4 个。

说实话，一开始看 2 个空格感觉挺难受的，但是慢慢地发现前端开发地缩进实在是太频繁了，一层一层像套娃一样。如果不用 2 个空格，代码宽度会很大，影响阅读。

7.1 使用方式

7.2 结合 radio

7.3 结合 checkbox

复选框分为两种情况：单个勾选框和多个勾选框。

  • 单个勾选框，v-model即为布尔值，此时 input 的 value 并不影响v-model的值。
  • 多个复选框，因为可以选中多个，所以对应的data中属性是一个数组。当选中某一个时，就会将 input 的 value 添加到数组中。

7.4 结合 select

和 checkbox 一样，select 也分单选和多选两种情况。

  • 单选，v-model绑定的是一个值。当选中 option 中的一个时，会将它对应的 value 赋值到 data 中
  • 多选，v-model绑定的是一个数组。当选中多个值时，就会将选中的 option 对应的 value 添加到数组中

7.5 修饰符

  • lazy修饰符：

    默认情况下，v-model默认是在 input 事件中同步输入框的数据的。也就是说，一旦有数据发生改变对应的data中的数据就会自动发生改变。lazy修饰符可以让数据在失去焦点或者回车时才会更新。

  • number修饰符：

    默认情况下，在输入框中无论输入的是字母还是数字，都会被当做字符串类型进行处理。但是如果我希望处理的是数字类型，那么最好直接将内容当做数字处理。number修饰符可以让在输入框中输入的内容自动转成数字类型。

  • trim修饰符：

    如果输入的内容首尾有很多空格，通常会将其去除。trim修饰符可以过滤内容左右两边的空格。
